/*
 ********************************************************************************
 *                          "PROJETO NÃO-MAGALU"                                *
 ********************************************************************************
Como falado na descrição, a ideia inicial do projeto consiste no desenvolvimento de
um sistema de gestão de dados de uma varejista
(o nome está para ser definido mas por enquanto será ”Não-Magalu”).
Esse sistema será elaborado em linguagem C, e conterá
(pelo menos pretende-se conter) CRUD’s dos produtos e fornecedores da loja.
Além disso, cada um desses CRUD’s deverá conter também uma opção para se ver
um relatório geral dos dados obtidos.
**********************************************************************************

*****************************************
*   ESTRUTURAS A SEREM UTILIZADAS       *
*****************************************
Criaremos tipos de estruturas para os seguintes itens:

1. Produtos:

--Nome do produto;
--Código do produto;
--Valor de venda;
--Pequena descrição do produto que está sendo vendido;
--Quantidade de itens em estoque.
--Valor lógico de exclusão


2. Fornecedores:

--Nome do fornecedor;
--Código do fornecedor;
--CNPJ;
--Endereço;
--Telefone;
--Produto que é fornecido;
--Preço de compra;
--Quantidade de produtos que são comprados;
--Descrição do produto;
--Data da última compra;
--Data de início das relações;
--Data do fim das relações;
--Valor lógico de exclusão;

3. Estrutura para verificar a exclusão

-- Código (itens nos arquivos)
-- Posição

**************************************************************************************


*************************************
*    UTILIZAÇÂO DAS ESTRUTURAS      *
*************************************

As estruturas de Produtos e Fornecedores serão utilizadas para:

O armazenamento dos dados dos Produtos disponives, e
dos Fornecedores da varejista, em arquivos expecíficos de cada para tipo de dado
(fornecedores e produtos).

Já a estrutura para verificação da exclusão será utilizada para:

--Para a maior eficiência do código, sendo que, ela aramazenará o código do
Produto ou Fornecedor, e sua posição dentro do arquivo onde estão armazenados
cada tipo de dado.

************************************
*   FUNCIONALIDADES DA APLICAÇÃO   *
************************************

##Descoberta do SO (para a limpeza da tela):

Primeiro menu que deve aparecer no programa a ser rodado,
pedindo para o usuário selecionar o SO utilizado (UNIX / WINDOWS).

Devido a diferença entre os sistemas, a limpeza de tela ocorre por meio de funções
distintas. Com isso, o resultado obtido por essa função irá definir o modo como a
limpeza de tela acontece.

system("cls") - Para sistemas Windows.
system("clear") - Para sistemas UNIX.


##Função para a limpeza de tela, de acordo com a opção dada na descoberta do sistema
operacional:

Está função limpa a tela com o resultado obtido pela função de seleção do Sistema
operacional.

##Menu principal:

O menu principal terá como objetivo a seleção de ações a ser realizadas, sendo elas:

        --------------------------------------
        * Escolha uma das seguintes opçõess: *
        * 1 - Gerenciar Produtos             *
        * 2 - Gerenciar Fornecedores         *
        * 3 - Sair                           *
        --------------------------------------

*A opção 1 leva para o menu de gerenciamento de Produtos;
*A opção 2 leva para o menu de gerenciamento de Fornecedores;
*A opção 3 finaliza a execução do programa.

##Menu para gerenciar os Produtos

O menu para gerenciar os produtos terá como objetivo a seleção de ações a serem
realizadas, sendo elas:

        --------------------------------------
        * Escolha uma das seguintes opçõess: *
        * 1 - INSERIR os produto             *
        * 2 - PESQUISAR o produto            *
        * 3 - ATUALIZAR os produtos          *
        * 4 - REMOVER os produtos            *
        * 5 - APRESENTAR RELATÓRIO GERAL     *
        * 6 - Sair                           *
        --------------------------------------

*A opção 1 leva para a função inserção de produtos;
*A opção 2 leva para a função pesquisa de produtos;
*A opção 3 leva para a função atualização de produtos;
*A opção 4 leva para a função remoção de produtos;
*A opção 5 leva para a função apresentação do relatório geral dos produtos;
*A opção 6 retorna ao MENU PRINCIPAL.

##Função para INSERIR os produtos

Primeiramente, ela abre o arquivo binário de produtos, caso não exista ela
cria um novo, e caso exista, ela abre para a escrita no final do arquivo.

Em seguida, é feita a limpeza da tela, e pede para o usuário digitar os
dados presentes na Struct de Produtos. Sendo os dados:

--Nome do produto;
--Código do produto;
--Valor de venda;
--Pequena descrição do produto que está sendo vendido;
--Quantidade de itens em estoque;
--Inserindo também no campo de valor lógico a indicação de que não está excluido.

Por fim, a função insere os dados obtidos no arquivo de Produtos, fechando o arquivo de produtos.


##Função para PESQUISAR os produtos

A função abre o arquivo binário de produtos para a leitura aleatória,
caso não exista o arquivo ela retorna uma mensagem de erro.

Em seguida, é feita a limpeza da tela, e pede para o usuário digitar o
código do Produto a ser pesquisado.

Em seguida é pesquisado o código do Produto dentro do vetor de posições,
caso encontre o código e o produto não esteja excluído logicamente,
ele abre o arquivo de produtos na posição exata dos dados do produto,
e os imprime na tela. Caso o código não seja encontrado, é apresentada
uma mensagem indicando que o produto não está cadastrado.

Por fim, é fechado o arquivo de produtos.


##Função para ATUALIZAR os produtos

A função abre o arquivo binário de produtos para a leitura e escrita aleatória,
caso não exista o arquivo ela retorna uma mensagem de erro.

Em seguida é pesquisado o código do Produto dentro do vetor de posições,
caso encontre o código e o produto não esteja excluído logicamente,
ele abre o arquivo de produtos na posição exata dos dados do produto,
e os imprime na tela, em seguida pede a confirmação se deseja atualizar os
dados apresentados. Assim, é feita a limpeza da tela, e pede para o usuário digitar os
dados presentes na Struct de Produtos. Sendo os dados:

--Nome do produto;
--Código do produto;
--Valor de venda;
--Pequena descrição do produto que está sendo vendido;
--Quantidade de itens em estoque;
--Inserindo também no campo de valor lógico a indicação de que não está excluido.

Entretanto se o código não for encontrado, é apresentada
uma mensagem indicando que o produto não está cadastrado.

Por fim, é fechado o arquivo de produtos.


##Função para REMOVER os produtos

A função abre o arquivo binário de produtos para a leitura e escrita aleatória,
caso não exista o arquivo ela retorna uma mensagem de erro.

Em seguida é pesquisado o código do Produto dentro do vetor de posições,
caso encontre o código e o produto não esteja excluído logicamente,
ele abre o arquivo de produtos na posição exata dos dados do produto,
e os imprime na tela, em seguida pede a confirmação se deseja remover os
dados apresentados. Assim, é feita ocorre a exclusão lógica do cadastro do produto
trocando o valor logico da variável "valor lógico de exclusão".

Entretanto se o código não for encontrado, é apresentada
uma mensagem indicando que o produto não está cadastrado.

Por fim, é fechado o arquivo de produtos.


##Função para apresentar o relatório geral dos dados dos produtos

A função abre o arquivo binário de produtos para a leitura sequencial,
caso não exista o arquivo ela retorna uma mensagem de erro.

Em seguida é apresentado todos os dados presentes no arquivo.

Por fim, é fechado o arquivo de produtos.

##Função para criar os vetores de verificação de exclusão

A função abre o arquivo binário de fornecedores ou produtos para a
leitura sequencial, caso não exista o arquivo ela retorna uma mensagem de erro.

Em seguida, é feita a leitura de todos os elementos do arquivo, coletando a
quantidade de itens dentro do arquivo.

Após isso, é alocado dinamicamente um vetor de Structs para Verificar a Exclusão,
com a quantidade de elementos do arquivo.

Por fim, é feito o armazenamento do código do produto ou fornecedor e sua posição
no arquivo.

fechando o arquivo aberto.


##Função para confirmação (Atualização / Remoção)

Esta função irá a confirmação se o usuário deseja de fato atualizar ou remover
o item selecionado.


##Função para a quantidade de elementos (que auxiliará na criação dos vetores da
função acima)

Esta função é a que faz a leitura de todos os elementos do arquivo, coletando a
quantidade de itens dentro do arquivo. Sendo chamada na função de criação dos
